# AGENTS.md

This file provides guidance to Codex (Codex.ai/code) when working with code in this repository.

## Running the App

```bash
python main.py
```

Default credentials: `admin` / `admin123`, `cashier` / `cash123`

## Installing Dependencies

```bash
pip install -r requirements.txt
```

Core packages: `customtkinter>=5.2.0`, `Pillow>=10.0.0`, `openpyxl>=3.1.0`, `reportlab>=4.0.0`

Optional (thermal printer): `python-escpos`, `pywin32`

## Building the Executable

```bash
python run_build.py
```

This runs PyInstaller with `PriyaStore.spec`, then copies `billing_data.db` and creates the `backups/` directory in `dist/PriyaStore/`. Output lands in `dist/PriyaStore/`.

Alternatively, run PyInstaller directly:
```bash
pyinstaller PriyaStore.spec --noconfirm
```

> `KunalBilling.spec` and `billing.spec` are older/unused specs. `database_fixed.py` and `main_fixed.py` are stale intermediate files — do not use them.

## Architecture

### Entry Point & Navigation

`main.py` → `BillingApp(ctk.CTk)` is the root window. After login it builds a sidebar + `content_area` frame. Navigation is handled by `navigate_to(screen_name)`, which lazily instantiates screens on first visit and caches them in `self.screens`. Screens are shown/hidden by pack/pack_forget — they are **never destroyed** between visits unless `rebuild_screen()` is called explicitly. Each screen receives `(parent, db, current_user, app)` on construction and must implement `on_show()` for data refresh.

### Screen Construction Contract

Every screen (e.g. `screen_billing.py`) follows the same pattern:
- Constructor: build all widgets once
- `on_show()`: reload data from DB and refresh the UI
- Call `self.app.navigate_to(...)` or `self.app.rebuild_screen(...)` to cross-navigate

### Database Layer (`database.py`)

Raw `sqlite3` — no ORM. All queries live in `Database`. Each method opens a fresh connection via `get_conn()`, which enables WAL mode and foreign keys. Key invariants:
- `save_bill()` deducts stock atomically in the same transaction
- `void_bill()` restores stock atomically
- `save_purchase()` increases stock and updates `purchase_price` atomically
- Bill numbers are generated by `next_bill_number()` then incremented by `increment_bill_number()` **after** the bill is committed — never inside the bill transaction
- Schema migrations are applied via `try/except` ALTER TABLE statements inside `init_db()` — add new columns there

DB path: `billing_data.db` next to the script (or next to the `.exe` when frozen). `config.DB_PATH` resolves this correctly for both environments.

**Tables:** `users`, `categories`, `customers`, `products`, `bills`, `bill_items`, `settings`, `activity_log`, `suppliers`, `purchase_entries`, `purchase_items`, `stock_adjustments`, `customer_transactions`, `supplier_payments`

### Activity Logging

Call `db.log_activity(user_id, action, details)` after every meaningful state change. Standard action strings (used by `screen_activity_log.py` for colour-coding): `LOGIN`, `LOGOUT`, `BILL_SAVED`, `BILL_VOIDED`, `GRN_SAVED`, `PRODUCT_*`, `USER_*`.

### Config & Styling

- **`config.py`** — single source of truth for `COLORS`, `FONTS`, `RADII`, `APP_TITLE`, `SHOP_NAME`, `WINDOW_WIDTH/HEIGHT`, `SIDEBAR_WIDTH`, `UNITS`, `PAYMENT_MODES`, `SHORTCUTS`. Never hardcode colors or font sizes in screen files.
- **`styles.py`** — `setup_ttk_styles()` registers every named `ttk.Treeview` and `TScrollbar` style once at startup. Each screen references its pre-registered style by name (e.g. `"Bill.Treeview"`). Never create a new `ttk.Style()` inside a screen.
- **`config.resource_path(*parts)`** — use this for any asset path (icons, images) so it works both in source and PyInstaller builds.

### Role-Based Access

Three roles: `admin`, `cashier`, `stock_manager`. Sidebar filters nav items by `self.current_role`. Screens themselves do not enforce access — the sidebar is the gatekeeper.

The NAV list in `main.py:_build_sidebar` is the authoritative mapping of screen keys to allowed roles.

### Bill Printing (`bill_printer.py`)

- `generate_pdf_bill(bill, items, settings)` → ReportLab A4 PDF, saved to a temp file, then opened with the OS default viewer
- Thermal ESC/POS printing is optional — guarded by try/import

### Backup

`screen_settings._run_backup(db)` copies `billing_data.db` into `backups/` with a timestamp. Called on app close and every 24 hours via `BillingApp._schedule_daily_backup()`. The `auto_backup_enabled` setting controls whether the timer actually writes a file.

## UI Conventions

- Font sizes: `FONTS["body"]=16`, `FONTS["heading"]=27`. Keep large for 60+ users.
- Sidebar: deep navy gradient (`#1E3A8A` → `#0F172A`), drawn with a `tk.Canvas` gradient loop.
- Cards: white with `COLORS["glass_border"]` borders — glassmorphism aesthetic.
- Tables: all use `ttk.Treeview` with 48px row height. Always use the screen-specific named style.
- Shop name is always "Priya Store" — enforced on every startup via `db.set_setting("shop_name", "Priya Store")`.
- Billing screen keyboard shortcuts: `F2` search product, `F8` hold bill, `F10` print & save, `Del` remove item, `Esc` clear cart.
